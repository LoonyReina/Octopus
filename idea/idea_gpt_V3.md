# OpenClaw 边缘开发板 + Docker 开发流程总结

## 目标

在 **边缘开发板 + Docker** 环境下开发 OpenClaw，做到：

- 本机和远端宿主机各保留一份源码
- 远端宿主机通过 Docker 提供运行环境
- 容器挂载远端源码目录
- 修改源码后尽快生效
- 可通过 **SSH + Claude Code** 远程开发

---

## 推荐总体方案

采用下面的模式：

- **本机**：保留一份源码，用于同步、备份、提交 Git
- **远端宿主机**：也保留一份源码，作为实际开发目录
- **Docker 容器**：使用 OpenClaw 官方镜像作为基础运行环境
- **容器运行时挂载远端源码目录**
- **通过 SSH + Claude Code 直接改远端宿主机上的代码**

核心原则：

> **代码在宿主机，环境在容器里**

---

## 为什么不建议直接在容器里改代码

虽然可以 `docker exec -it <container> bash` 进去直接改，但不推荐作为正式开发方式，因为：

- 容器删除后改动可能丢失
- 重建容器后不方便复用
- 不利于 Git 管理
- 不利于远程协作和长期维护

因此更推荐：

- **源码放宿主机**
- **容器只负责依赖、构建、运行**

---

## OpenClaw 场景下要特别注意的点

OpenClaw 官方镜像通常已经内置好了 `claw`。

但如果你要修改自己的源码，必须注意：

> **容器最终运行的必须是你挂载进去并构建后的代码，而不是镜像里预装的全局 `claw`。**
